# Mnemosyne

🧠 MnemosyneThe Invisible Flight Recorder for AI-Assisted DevelopmentMnemosyne (n.) From Greek mythology, the Titaness of Memory. A deterministic, out-of-band telemetry daemon designed to solve AI State Loss and Context Collapse.🔴 The Problem: "AI Amnesia" & Context CollapseWhen developers utilize agentic AI tools (Claude, GitHub Copilot, ChatGPT, Antigravity) for deep, multi-file refactoring, they build a massive, shared "working context" with the model.When a strict API rate limit, usage cap, or token exhaustion occurs, this continuous flow is severed. The developer is locked out, and the AI's active memory freezes. Switching to a fresh AI session normally requires manually copying and pasting massive file trees, re-explaining rejected iterations, and re-establishing architectural rules.Current state-of-the-art (SOTA) workarounds possess catastrophic blind spots:The Failure of In-Band Memory (MemGPT / Mem0): Current memory agents operate in-band, relying on the LLM to manage its own memory via tool-calling. When a rate limit HTTP 429 strikes, the LLM runtime crashes synchronously, taking the memory manager down before it can save the state."Lost in the Middle" Degradation: Simply copy-pasting a full project into a new prompt fails. As proven by Liu et al., Transformer self-attention mechanisms exhibit a severe U-shaped spatial degradation. Critical code placed in the middle of a massive prompt is mathematically diluted and ignored by the attention weights.Latent Reasoning Collapse: The NoLiMa (No Linguistic Markers) benchmark proves that standard models suffer a catastrophic drop in reasoning capabilities (up to 59.8% decline) at just 32K tokens when trying to track complex codebase architectures.🟢 Our SolutionMnemosyne is a strictly local, out-of-band background daemon. It operates independently of any AI provider.It acts as an invisible "Flight Recorder," silently tracking file modifications, inter-file dependencies, terminal errors, and developer intent into a local, high-speed SQLite graph database. When an AI lockout occurs, Mnemosyne uses a deterministic mathematical algorithm to compress the last active, un-rejected state of the project into a hyper-dense Handoff Manifest.This allows you to open a brand new chat window and instantly inject perfect context, bypassing the 15-to-30 minute cognitive recovery penalty associated with hard task interruptions.✨ Core Features & Implementation Details1. Invisible Background Telemetry & Causal GraphingMnemosyne doesn't just track single files; it builds a multi-file causal graph of your development trajectory.Implementation: A Node.js/TypeScript daemon utilizes chokidar to monitor file saves. It spawns ephemeral WebAssembly (web-tree-sitter) workers to parse ASTs and extract import/require statements, dynamically mapping cross-file DEPENDS_ON edges.Terminal Interception: It wraps standard terminal streams (node-pty) using OS-specific backpressure handling (POSIX strict on UNIX, dynamic buffer queues on Windows). It maps both stderr failures and stdout_success (exit code 0) to track which code variations actually worked.2. State-Reality Synchronization & IsolationThe system maintains absolute parity between your codebase and your graph without cross-contamination.Implementation: Mnemosyne utilizes Workspace Sandboxing. It binds session identity using a workspace_hash and terminal PID to prevent multiple IDE windows from corrupting the SQLite ledger.Shadow Rollback Verification: When you run mnemo reject to undo a failed experiment, the daemon diffs the current filesystem against a compressed shadow state. If the rejected code is still present in your IDE, it throws a fatal warning to ensure graph and reality never diverge.3. The Super-Compressor (Knapsack Manifest Compiler)When a lockout occurs, Mnemosyne calculates a strict token budget bounded by the provider's Prefix Caching boundaries ($B_{prefix}$) and maximum context size ($C_{max}$) via the formula: $K_{tokens} = \min(B_{prefix}, \gamma \cdot C_{max})$.Implementation: The Python optimization engine solves the context selection using a Bounded 0/1 Knapsack algorithm (executed via Dantzig's Greedy approximation in Sprint 1, upgrading to a formal $O(n^2 \cdot \frac{1}{\epsilon})$ FPTAS Dynamic Programming matrix in Sprint 2). It ranks active graph nodes by causal centrality and recency to pack the absolute most critical data into the handoff manifest.4. Memory Anchor (Attention Injection)To combat Transformer "attention dilution," Mnemosyne structures the output mathematically.Implementation: It isolates unbreakable project rules (e.g., // VCS: constraint[db]=use_postgres). If conflicting rules exist, it applies a Last-Writer-Wins (LWW) resolver. It injects the winning constraints as an Active Constraints Header at the extreme absolute top of the Handoff Manifest, explicitly exploiting the model's primacy attention sink.5.
